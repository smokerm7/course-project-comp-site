# Спецификации прецедентов

## UC1. Авторизация пользователя

| Поле | Значение |
|---|---|
| Актор | Клиент, сотрудник, администратор |
| Цель | Получить доступ к защищённым функциям |
| Предусловие | Пользователь зарегистрирован |
| Результат | Пользователь получает JWT-токен |

Основной поток: пользователь вводит email и пароль, система проверяет данные, формирует JWT и возвращает профиль пользователя.

## UC2. Оформление заказа

| Поле | Значение |
|---|---|
| Актор | Клиент |
| Цель | Купить товары из каталога |
| Предусловие | Клиент авторизован, корзина не пуста |
| Результат | Создан заказ со статусом «Новый» |

Основной поток: клиент выбирает товары, добавляет в корзину, подтверждает заказ, система проверяет остатки, создаёт заказ и уменьшает складские остатки.

## UC3. Создание заявки на ремонт

| Поле | Значение |
|---|---|
| Актор | Клиент |
| Цель | Передать информацию о неисправном устройстве |
| Предусловие | Клиент авторизован |
| Результат | Создана заявка со статусом «Принята» |

Основной поток: клиент указывает тип устройства, модель и описание неисправности, система сохраняет заявку и отображает номер заявки.

## UC7. Изменение статуса ремонта

| Поле | Значение |
|---|---|
| Актор | Сотрудник |
| Цель | Обновить этап выполнения ремонта |
| Предусловие | Сотрудник авторизован |
| Результат | Клиент видит новый статус ремонта |

Основной поток: сотрудник открывает заявку, выбирает новый статус, добавляет комментарий, система сохраняет изменения.
