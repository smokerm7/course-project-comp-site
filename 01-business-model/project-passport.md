# Паспорт проекта

| Параметр | Значение |
|---|---|
| Название | MS7-COMP |
| Полное название | Веб-приложение для продажи и ремонта компьютерной техники |
| Тип | Клиент-серверное веб-приложение |
| Предметная область | Компьютерная фирма: продажи, заказы, ремонт |
| Траектория | Веб-разработка |
| Frontend | React 18, TypeScript, Vite, Axios |
| Backend | Java 17, Spring Boot 3.x |
| База данных | PostgreSQL |
| Архитектура | PCMEF |

## Проблема

В небольшой компьютерной фирме заказы, ремонтные заявки и данные клиентов часто ведутся в мессенджерах, таблицах или бумажных журналах. Такой подход приводит к потере информации, ошибкам в остатках товаров и отсутствию прозрачного контроля ремонта.

## Решение

Разработать веб-приложение, которое объединит каталог товаров, оформление заказов, заявки на ремонт, личный кабинет клиента и административную панель.

## Цели

1. Автоматизировать продажу компьютерной техники и комплектующих.
2. Автоматизировать приём и обработку заявок на ремонт.
3. Создать единую базу клиентов, товаров, заказов и ремонтных заявок.
4. Реализовать отслеживание статусов заказов и ремонта.
5. Обеспечить разграничение доступа по ролям.

## KPI

| Показатель | Целевое значение |
|---|---|
| Время оформления заказа | до 2 минут |
| Время создания заявки на ремонт | до 1 минуты |
| Количество ролей | 3 |
| Количество основных сущностей БД | не менее 9 |
| Количество REST API эндпоинтов | не менее 15 |
| Покрытие базовых сервисов тестами | от 40% |

## Риски

| Риск | Вероятность | Меры реагирования |
|---|---:|---|
| Слишком широкий функционал | Средняя | Реализовать MVP: каталог, заказ, ремонт, админка |
| Ошибки проектирования БД | Средняя | Сначала подготовить ER-модель и DDL |
| Сложность авторизации | Средняя | Использовать Spring Security + JWT |
| Недостаток времени на UI | Высокая | Сделать простой адаптивный интерфейс |
